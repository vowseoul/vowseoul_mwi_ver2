## 2. 모바일 청첩장 테마 제작 및 커스터마이징
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

테마 디자이너가 외부 툴(피그마 등)에서 디자인한 HTML/CSS 파일을 업로드하여 새로운 모바일 청첩장 테마로 등록할 수 있습니다. 등록 시 특정 사진 및 텍스트 영역을 데이터 필드와 연결(매핑)할 수 있으며, 색상, 폰트, 레이아웃 등 세밀한 디자인 요소를 시스템 내에서 수정할 수 있습니다.

**수용 기준:**
```
1. ○ 디자이너는 HTML, CSS 파일을 업로드하여 새로운 테마를 등록할 수 있어야 합니다.
2. ○ 테마 등록 시, 이미지 및 텍스트 영역에 지정된 규칙(예: main_image1, invitation_message)에 따라 데이터 필드를 매핑할 수 있어야 합니다.
3. ○ 등록된 테마에 대해 색상, 폰트 스타일, 패딩, 위치 등을 웹 인터페이스를 통해 시각적으로 수정할 수 있어야 합니다.
4. ○ 레이아웃 및 섹션 순서 변경 기능을 제공해야 합니다.
```

디자이너가 HTML/CSS 테마를 업로드해 테마로 등록하고, 수정 시 버전을 추가로 발행할 수 있습니다. 운영자는 고객 청첩장에 적용할 테마/버전을 선택할 수 있으며, 기존 고객 청첩장에는 영향을 주지 않도록 버전 고정을 지원합니다.


#### 2.1.1 테마 파일 업로드 검증(필수 파일/규칙)

업로드 시 필수 파일 존재 여부(예: index.html, style.css)와 금지 리소스/외부 참조 규칙을 검사하고, 실패 사유를 표시합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 테마 디자이너 | **기기**: Web


#### 2.1.2 테마 미리보기 및 샘플 데이터 렌더링

샘플 데이터로 테마를 렌더링해 미리보기 합니다. 주요 섹션별(핵심 정보/갤러리/오시는 길/RSVP/방명록) 렌더링 상태를 확인합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 테마 디자이너 | **기기**: Web


### 2.2 테마 영역-데이터 필드 매핑(바인딩) 설정
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 테마 디자이너 | **기기**: Web

테마 내 텍스트/이미지/리스트 영역을 데이터 필드에 연결(매핑)합니다. 예: invitation_message ↔ 초대 인사말, main_image1 ↔ 메인 이미지. 매핑 누락/중복을 검증하고 미리보기에서 바인딩 결과를 확인할 수 있습니다.


#### 2.2.1 매핑 키 자동 탐지 및 추천

테마 내 지정된 규칙(예: data-key 속성, placeholder 키)을 탐지해 매핑 후보를 추천합니다. 운영자는 추천을 선택해 빠르게 바인딩을 완료할 수 있습니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 테마 디자이너 | **기기**: Web


#### 2.2.2 매핑 검증(누락/중복/타입 불일치)

매핑 누락, 동일 필드의 중복 매핑, 타입 불일치(예: 이미지 영역에 텍스트 필드 연결)를 검증하고 오류 목록을 제공합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 테마 디자이너 | **기기**: Web

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 테마 디자이너, VOW SEOUL 운영자 | **기기**: Web

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 테마 디자이너, VOW SEOUL 운영자 | **기기**: Web

## 3. 모바일 청첩장 자동 생성 및 수정
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

고객이 제출한 정보 수집 폼 데이터를 바탕으로, 선택된 모바일 청첩장 테마에 자동으로 정보를 채워 초안을 생성합니다. 운영자는 이 초안에 대해 사진 업로드, 텍스트 수정, 배경색 변경 등 상세한 수정 작업을 수행할 수 있습니다.

**수용 기준:**
```
1. ○ 선택된 고객의 폼 데이터가 선택된 테마에 자동으로 적용되어 모바일 청첩장 초안이 실시간으로 생성되어야 합니다.
2. ○ 운영자는 청첩장 내 사진을 업로드, 변경, 삭제할 수 있어야 합니다.
3. ○ 운영자는 텍스트 내용, 폰트, 색상 등을 직접 수정할 수 있어야 합니다.
4. ○ 실시간 미리보기 기능을 제공하여 수정 사항을 즉시 확인할 수 있어야 합니다.
```


### 3.1 고객 선택 → 테마 선택 → 초안 자동 생성

운영자가 고객을 선택하고 테마/버전을 선택하면, 제출된 폼 데이터가 매핑 규칙에 따라 자동으로 적용되어 초안이 생성됩니다. 초안 생성 시 누락 데이터/매핑 실패 항목을 표시하고 보완을 안내합니다.

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: VOW SEOUL 운영자 | **기기**: Web

운영자가 초안에서 텍스트와 이미지(메인 이미지, 갤러리)를 수정하고 즉시 미리보기로 확인합니다. 편집 변경 이력(되돌리기/다시하기)을 제공하고, 모바일 화면 기준으로 미리보기를 지원합니다.

## 4. 청첩장 발행 및 고객 대시보드
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

최종 완성된 모바일 청첩장을 발행하면, 하객들에게 공유할 수 있는 청첩장 링크와 결혼 고객(신랑 신부)이 청첩장 현황을 관리할 수 있는 대시보드 링크가 각각 발급됩니다. 대시보드에서는 방문자 수, RSVP 응답, 방명록 조회 및 관리 기능이 제공됩니다.

**수용 기준:**
```
1. ○ 모바일 청첩장 발행 시 고유한 청첩장 링크가 생성되어야 합니다.
2. ○ 결혼 고객을 위한 별도의 관리 대시보드 링크가 생성되어야 합니다.
3. ○ 고객 대시보드에서는 청첩장 링크 방문자 수 통계를 확인할 수 있어야 합니다.
4. ○ RSVP 응답 내역을 조회하고 관리할 수 있어야 합니다.
5. ○ 방명록 댓글을 조회하고 특정 글을 비공개 처리할 수 있어야 합니다.
```

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: VOW SEOUL 운영자 | **기기**: Web

청첩장을 발행하면 하객용 청첩장 URL과 고객용 관리 대시보드 URL을 생성합니다. 발행 상태(초안/발행/일시중지)를 관리하고, 링크 재발급(필요 시)과 공유용 복사를 지원합니다.


#### 4.1.1 발행 상태 관리(초안/발행/일시중지)

청첩장의 발행 상태를 관리합니다. 일시중지 시 하객 링크 접근 시 안내 화면을 노출합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: VOW SEOUL 운영자 | **기기**: Web

> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: VOW SEOUL 운영자 | **기기**: Web

방문자 수를 일/주/누적 기준으로 보여주며 대시보드 화면에서 그래프로도 볼 수 있습니다.

## 5. 모바일 청첩장 핵심 기능 구현 (MVP)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

발행되는 모바일 청첩장은 신랑신부 및 혼주 정보, 예식 일시 및 장소, 초대 인사말, 메인 이미지, 오시는 길(지도 및 대중교통 안내) 등 필수적인 기능을 포함해야 합니다.

**수용 기준:**
```
1. ○ 신랑신부 및 혼주 정보 섹션이 정확히 표시되어야 합니다.
2. ○ 예식 일시, 장소(주소, 상세 지도 연동) 정보가 명확하게 제공되어야 합니다.
3. ○ 초대 인사말을 입력하고 표시할 수 있어야 합니다.
4. ○ 메인 이미지 및 포토 갤러리 기능이 제공되어야 합니다.
5. ○ 오시는 길 섹션에서 지도 연동 및 대중교통 안내 정보가 제공되어야 합니다.
```


### 5.1 핵심 정보 섹션 표시 및 구성
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: VOW SEOUL 운영자 | **기기**: Mobile

모바일 청첩장에 반드시 포함되어야 하는 핵심 정보를 섹션 단위로 구성하여 표시합니다. 신랑·신부/혼주/예식 정보, 초대 인사말, 메인 이미지, 오시는 길(지도·대중교통) 안내를 기본 섹션으로 제공하며, 테마에 맞춰 노출 순서와 표시 여부를 제어할 수 있습니다.


#### 5.1.1 핵심 정보 섹션: 신랑·신부/혼주 정보 표시

신랑·신부 이름, 연락처(선택), 혼주 호칭/이름/연락처(선택)를 지정된 레이아웃으로 표시합니다. 노출 항목(연락처 등)은 켜기/끄기를 지원합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: VOW SEOUL 운영자 | **기기**: Mobile


#### 5.1.2 핵심 정보 섹션: 예식 일시/장소 표시

예식 일시, 장소명, 주소를 표시하고, 주소는 지도 섹션과 일관된 값으로 유지됩니다(단일 소스).
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: VOW SEOUL 운영자 | **기기**: Mobile


#### 5.1.3 핵심 정보 섹션: 초대 인사말 표시

초대 인사말을 입력하고 청첩장에 표시합니다. 줄바꿈과 기본 서식(강조/정렬 등 제한적 옵션)을 지원합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: VOW SEOUL 운영자 | **기기**: Mobile


#### 5.1.4 핵심 정보 섹션: 메인 이미지 표시

메인 이미지를 1장 설정하고 청첩장 상단에 표시합니다. 기기별(모바일) 화면 비율에 맞춰 자동 크롭/리사이즈 옵션을 제공합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: VOW SEOUL 운영자 | **기기**: Mobile


#### 5.1.5 오시는 길: 지도 연동 및 대중교통 안내 표시

오시는 길 섹션에서 지도 연동(내장 지도 또는 외부 지도 열기)과 대중교통 안내 문구를 표시합니다. 안내 문구는 운영자가 편집할 수 있습니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: VOW SEOUL 운영자 | **기기**: Mobile


#### 5.1.6 카카오톡 공유 메세지 설정

카카오톡에 링크 공유 시 뜨는 썸네일과 텍스트 (타이틀과 설명글로 구성)를 설정할 수 있음.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중


### 5.2 추가 기능 섹션 표시 및 구성
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: VOW SEOUL 운영자, 테마 디자이너 | **기기**: Web

청첩장 기능(섹션)을 ‘블럭’ 단위로 제공해, 운영자가 청첩장별로 필요한 블럭을 추가/제거하고 순서를 바꿔 커스텀할 수 있게 합니다. 기본 블럭(핵심 정보/오시는 길 등) 외에 포토 갤러리, 계좌 안내, 연락하기, 공유, 영상, FAQ 등 확장 블럭을 제공하며, 테마별로 허용 블럭과 기본값(기본 포함/옵션)을 관리합니다. 블럭별로 데이터 필드 연결, 표시 조건, 스타일 옵션을 지원하고 미리보기에서 즉시 확인합니다.


#### 5.2.1 추가기능: 포토 갤러리

좌우 슬라이드로 움직이는 슬라이드형, 혹은 아래로 스크롤 하며 볼 수 있는 그리드 형 중 선택 가능하며 최대 20장 첨부 가능. 확대 기능 제거.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중


#### 5.2.2 추가기능: RSVP 참석 의사 수집 

RSVP 버튼을 청첩장에 삽입하여 청첩장을 받은 하객들의 참석 여부/식사여부/셔틀이용여부 를 조사할 수 있음. 성함과 연락처, 참석여부와 신랑/신부측 구분 및 동행인 수를 기본으로 조사하며 사전에 설정시 식사 여부와 설정한 메뉴 중 선택하는 조사와 셔틀 운행 정보를 보여주고 이용여부도 조사 가능. 해당 기능 사용시 조사 내역은 고객 대시보드와 연동되어 결과 확인 가능
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중


#### 5.2.3 추가기능: 축의금 송금 계좌

신랑 신부 및 혼주의 계좌번호를 입력하여 노출시키고, 하객이 모바일 청첩장 화면에서 클릭 시 계좌번호 정보를 클립보드에 바로 복사하는 기능.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중


#### 5.2.4 추가기능: BGM 추가

모바일 청첩장 링크를 통해 접속 시 자동으로 설정한 BGM(MP3 파일)이 재생됨. 화면 우측 상단에 작은 플로팅 버튼을 통해 음악 소리를 끄고 킬 수 있음.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중


#### 5.2.5 블럭 라이브러리(섹션 카탈로그) 관리

관리자 화면에서 사용 가능한 블럭 목록을 제공합니다. 블럭별 이름/설명/미리보기(샘플)/필수 여부(기본 포함/옵션)/권장 위치 등을 포함합니다. 테마별 허용 블럭을 제한할 수 있습니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: VOW SEOUL 운영자, 테마 디자이너 | **기기**: Web


#### 5.2.6 청첩장별 블럭 추가/제거 및 순서 변경

청첩장 커스텀 화면에서 블럭을 추가/제거하고, 드래그로 순서를 변경합니다. 필수 블럭은 제거 불가로 처리하고, 제거/변경 시 미리보기와 발행본에 동일하게 반영됩니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: VOW SEOUL 운영자 | **기기**: Web


#### 5.2.7 블럭별 설정(데이터 바인딩/표시 조건/스타일 옵션)

블럭마다 연결할 데이터 필드(예: 계좌 블럭 ↔ 계좌 정보), 표시 조건(예: 값이 없으면 숨김), 스타일 옵션(간격/정렬/구분선 등 제한적)을 제공합니다. 설정 변경은 즉시 미리보기에 반영됩니다. 블럭별로 데이터 바인딩 설정을 끄고 킬 수 있으며 바인딩을 끌 시 직접 내용 입력도 가능
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: VOW SEOUL 운영자, 테마 디자이너 | **기기**: Web


#### 5.2.8 블럭 간 충돌/중복 규칙 및 검증

동일 블럭 중복 추가 허용 여부를 블럭별로 정의하고(예: 갤러리 1개만), 필수 블럭 누락, 테마 비호환 블럭 추가 등을 검증해 오류/경고로 안내합니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: VOW SEOUL 운영자, 테마 디자이너 | **기기**: Web
