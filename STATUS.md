## 2. 최근 변경 내역 (Recent Changes - Sprints 35 ~ 42)

### 2.1 루트 랜딩 스킵 & 대시보드 즉시 이동
* 루트 경로(`/`)에 존재하던 홍보용 luxury 랜딩 화면을 완전히 제거하고, Next.js `redirect('/admin')` 처리를 통해 관리자 포털 대시보드로 즉각 리다이렉트되도록 개선했습니다.

### 2.2 대시보드, 주문관리, 통계 페이지 실데이터 반영
* 기존의 하드코딩된 주문 리스트(`sampleOrders`) 및 차트용 상수 데이터를 완전히 걷어내고, Supabase `customers`, `invitations`, `visit_daily_stats` 테이블을 조합 및 연산하여 결제액 및 추이를 반영하는 완전한 실시간 대시보드를 구축했습니다.
* 주문 정보 수정 시 `customers` 테이블의 정보(신랑/신부 명, 예식일, 메모) 및 `invitations` 테이블의 활성화 상태가 실시간으로 연동 처리되도록 데이터 CRUD 함수를 구현했습니다.

### 2.3 `timentext` (식순 타임라인) 신규 필드타입 추가
* 모바일 청첩장의 식순 항목 수집을 위해 시간(시:분)과 식순 텍스트를 한 행으로 입력하는 신규 필드타입을 추가했습니다.
* **고객 입력**: 기본 3개 행 제공, 동적 추가(`+`) 및 삭제(`✕`) 가능.
* **어드민 목록**: 읽기 모드에서는 `"12:00 | 입장, 12:10 | 주례"` 처럼 복사 및 확인이 용이한 단일 문자열로 표기하고, 수정 모드에서는 상세 타임라인 편집 폼 제공.

### 2.4 제출 응답 상세 페이지의 복사 모드 & 원본 다운로드
* 어드민 응답 상세 페이지에서 기본 상태는 텍스트를 드래그하여 바로 복사할 수 있는 읽기 전용 모드를 제공하고, 수정하기 버튼을 클릭했을 때만 입력 폼이 노출되도록 개선하여 작업 편의성을 높였습니다.
* 고객이 제출한 첨부 파일(이미지)에 대해 Supabase Storage 주소의 원본 Blob을 다운로드해 주는 무손실 다운로드(`downloadOriginalImage`) 기능을 연동했습니다.

### 2.5 모바일 카카오톡 환경 최적화
* **localStorage 임시저장**: 인-앱 브라우저에서 입력 중 이탈이 잦은 특성을 고려해 매 키입력마다 `vowseoul_draft_{slug}`에 값을 자동 저장하고, 다시 진입 시 이어서 쓸 수 있도록 복구하는 안전장치를 추가했습니다.
* **BGM 오디오 연동**: 모바일 브라우저의 오디오 자동 재생 제한을 예방하기 위해 `preload="auto"`, `playsInline` 속성을 연동하고, Play Promise 예외 캐싱을 적용해 재생 안정성을 확보했습니다.
* **가이드 이미지 라이트박스**: 모바일 환경에서 안내 이미지가 너무 작게 표시되는 문제를 해결하기 위해 고해상도 확대 모달(`Lightbox`)을 구현했습니다.
